## Правила работы с базой (важно — безопасность)
- **Изменения схемы — только через файлы миграций**, не сырым SQL по живой базе:
  `supabase migration new <имя>` → пишем SQL в файл → `supabase db push`.
  Так изменения версионируются, обратимы и видны в git.
- **Supabase MCP подключать в режиме `--read-only`** и со `--project-ref=oqivrkghvpbybildfjtn`
  (только инспекция/запросы; запись — через миграции). Supabase — это боевая БД, ломать случайно нельзя.
- Перед деструктивными командами (`db reset`, `drop`, удаление данных) — спрашивать подтверждение.

## Полезные команды
- `supabase status` / `supabase projects list` — состояние
- `supabase migration new <имя>` — новая миграция
- `supabase db push` — применить миграции к облаку
- `supabase db pull` — стянуть текущую схему из облака
- Секреты (anon/service ключи, пароль БД) — НЕ коммитить; держать в `.env` (он в .gitignore).
